# Financial Services MCP

Financial Services domain intelligence MCP for banking, insurance, trading, payments, AML/KYC, open banking, and digital assets.

This implementation follows the shared domain MCP contract from `domain-mcp-specifications.md` and includes:

- Universal domain tools (`about`, `classify_data`, `assess_applicability`, `build_control_baseline`, etc.)
- Financial-specific tools (`classify_financial_entity`, `scope_pci_dss`, `assess_swift_csp`, `classify_digital_asset_service`)
- SQLite + FTS5 domain dataset with architecture patterns, data taxonomy, threat catalog, standards crosswalk, applicability rules, and evidence artifacts
- Financial-domain standards are curated here; generic cross-sector security baselines are delegated to the `security-controls` foundation MCP
- Ingestion pipeline with raw domain inputs in `ingestion/raw`, compiled dataset artifacts in `data/`, and provenance export in `sources.yml`
- EU + US baseline jurisdiction coverage generation during ingestion (27 EU member states + 50 US states + DC)
- Temporal obligation reasoning (`as_of_date`) with generated obligation graph nodes/edges
- State-specific US breach profile pipeline keyed for law MCP lookups with provenance tiers (`primary` statute vs `secondary` aggregation) and confidence levels
- Expert knowledge pack for high-risk financial domains:
  - instant-payment APP fraud
  - cloud control-plane concentration risk
  - sanctions screening evasion
  - model-risk/fair-lending drift
- Enhanced advisory outputs:
  - `build_control_baseline` now returns prioritized `risk_scenarios` linked to controls
  - `build_evidence_plan` now labels evidence relevance as `direct` vs `contextual` based on baseline regulation overlap
- Dual transport support:
  - `stdio` (MCP JSON-RPC framed with `Content-Length`)
  - HTTP JSON-RPC at `POST /mcp`
- Health endpoint at `GET /health` with status (`ok`/`degraded`/`stale`) and machine-readable reasons

## Requirements

- Node.js `>=25.0.0` (uses built-in `node:sqlite`)

## Run

```bash
npm start
```

Or run directly via package binary:

```bash
npx @ansvar/financial-services-mcp
```

By default both transports run:

- stdio enabled (`MCP_ENABLE_STDIO=true`)
- HTTP enabled on `127.0.0.1:3000` (`MCP_ENABLE_HTTP=true`)

Optional environment variables:

```bash
MCP_HTTP_HOST=0.0.0.0
MCP_HTTP_PORT=8080
MCP_ENABLE_STDIO=false
MCP_ENABLE_HTTP=true
```

## Data Ingestion

Bootstrap ingestion input files from embedded seed data:

```bash
npm run ingest:bootstrap
```

Build compiled dataset artifacts:

```bash
npm run ingest
```

Regenerate full US state breach override references (51 jurisdictions) before ingest when updating state breach baseline inputs:

```bash
node scripts/ingest/generate-us-state-breach-overrides.mjs
```

Validate drift between `ingestion/raw` and compiled dataset hashes:

```bash
npm run check:drift
```

Run data completeness/correctness gate (coverage + cross-reference integrity):

```bash
npm run check:data
```

`check:data` also enforces that all regulation IDs referenced across rules/threats/standards/evidence exist in `ingestion/reference/regulatory_catalog.eu-us.json`, that every catalog regulation is covered in applicability-to-standard mappings, standards crosswalks, threat scenarios, and evidence artifacts, that no standards/patterns/threats are orphaned (unused standards, patterns without threats, patterns without applicability rules, or threats without evidence linkage), that each `evidence_artifacts.template_ref` file exists, and that standards/evidence have cross-layer linkage (standards link to threats+evidence; evidence links to applicability rules) via shared regulation IDs.

Verify source URL reachability and capture HTTP/title health report:

```bash
npm run ingest:verify-sources
```

Artifacts:

- Raw inputs: `ingestion/raw/*.json`
- Coverage references: `ingestion/reference/*.json`
- US breach override references: `ingestion/reference/us_state_breach_overrides.json`
- Compiled dataset: `data/domain-dataset.json`
- Hash manifest: `data/dataset-hashes.json`
- Coverage report: `data/coverage-report.json`
- Source health report: `data/source-health.json` (generated by `ingest:verify-sources`)
- Source provenance registry: `sources.yml`

## Production Audit

Run MCP production audit checks aligned with the six-phase quality gate model:

```bash
npm run audit:production
```

Production audit now includes runtime smoke calls against core tools (`about`, `assess_applicability`, `get_obligation_graph`, `build_control_baseline`, `build_evidence_plan`, `compare_jurisdictions`) in addition to structural checks.
It also validates artifact integrity/freshness (fingerprint alignment, coverage alignment, and max artifact age threshold).

Recommended validation sequence for data quality first:

```bash
npm run ingest
npm run check:drift
npm run check:data
npm run check
npm test
```

## Notes

- Domain responses include standardized metadata:
  - citations
  - effective date
  - confidence
  - inference rationale
  - dataset version and fingerprint
  - out-of-scope notes
  - foundation MCP call routing hints
- Foundation MCP joins are represented as join metadata and call plans; wire live federation by connecting this MCP to foundation MCP endpoints.
- To resolve foundation calls directly from tool responses, pass `resolve_foundation: true` in tool arguments and configure:
  - `FOUNDATION_MCP_EU_URL`
  - `FOUNDATION_MCP_US_URL`
  - `FOUNDATION_MCP_CONTROLS_URL`
- If these are not set, the server uses production defaults:
  - `https://eu-regulations-mcp.vercel.app/mcp`
  - `https://us-regulations-mcp.vercel.app/mcp`
  - `https://security-controls-mcp.vercel.app/mcp`
- Temporal queries: pass `as_of_date` (`YYYY-MM-DD`) to `assess_applicability`, `assess_breach_obligations`, `compare_jurisdictions`, and `get_obligation_graph`.
- `as_of_date` is strict calendar-validated (invalid dates such as `2026-02-30` are rejected).
- Runtime prefers compiled ingestion data (`data/domain-dataset.json`) and automatically falls back to embedded seed data if the compiled file is missing or invalid.
